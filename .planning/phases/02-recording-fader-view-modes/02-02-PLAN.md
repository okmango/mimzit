**CompositorView.swift** â€” `struct CompositorView: UIViewRepresentable` per Pattern 2:

Parameters:
- `let playerLayer: AVPlayerLayer` â€” reference video (bottom sublayer)
- `let previewLayer: AVCaptureVideoPreviewLayer?` â€” camera preview (top sublayer, optional until CaptureEngine configures)
- `@Binding var videoBlend: Float` â€” 0.0 = reference only, 1.0 = camera only (FADER-01)

`makeUIView(context:)`:
- Create `UIView()` with `backgroundColor = .black`
- Set `playerLayer.videoGravity = .resizeAspectFill`
- `view.layer.addSublayer(playerLayer)`
- If previewLayer exists: set `.videoGravity = .resizeAspectFill`, `view.layer.addSublayer(previewLayer)`
- Return view

`updateUIView(_ uiView:, context:)`:
- Wrap ALL frame and opacity updates in `CATransaction.begin()` / `CATransaction.setDisableActions(true)` / `CATransaction.commit()` (Pitfall 3 â€” prevents implicit CALayer animations)
- `playerLayer.frame = uiView.bounds`
- If previewLayer exists and is not yet a sublayer, add it: `uiView.layer.addSublayer(previewLayer)` (handles late initialization)
- `previewLayer?.frame = uiView.bounds`
- `previewLayer?.opacity = videoBlend` (FADER-04: GPU-composited, zero CPU cost)

Add `.ignoresSafeArea()` when used to fill entire screen edge-to-edge (D-01).

**FaderView.swift** â€” `struct FaderView: View` per Pattern 5 and UI-SPEC FaderView component:

Parameters:
- `@Binding var value: Float` â€” 0.0 to 1.0
- `let trackHeight: CGFloat` â€” 8 for video fader, 4 for audio fader (UI-SPEC)
- `let leftLabel: String` â€” "REF" for both faders

Body â€” `GeometryReader`:
- **Track background:** `Capsule().fill(Theme.faderTrack).frame(height: trackHeight)` centered vertically
- **Filled track:** `Capsule().fill(Theme.faderFilled).frame(width: CGFloat(value) * trackWidth, height: trackHeight)` aligned leading
- **Thumb:** `Circle().fill(Color.white).frame(width: 28, height: 28).shadow(radius: 4, y: 2)` offset by `CGFloat(value) * (trackWidth - 28)`
- **Left label:** `Text(leftLabel).font(.system(size: 11)).foregroundColor(Theme.dimmedText)` positioned at leading edge
- **Right label:** `Text(rightLabel).font(.system(size: 11)).foregroundColor(Theme.dimmedText)` positioned at trailing edge
- Entire view `.frame(height: 44)` minimum touch target (UI-SPEC)

Gesture â€” `DragGesture(minimumDistance: 0)`:
- `.onChanged`: compute `newValue = Float(drag.location.x / trackWidth).clamped(to: 0...1)`. For haptic snap (D-09): check if `newValue` crosses within 0.02 of snap points `[0.0, 0.5, 1.0]` while `value` was outside â€” trigger `UIImpactFeedbackGenerator(style: .light).impactOccurred()`. Set `value = newValue`. Set isDragging = true.
- `.onEnded`: set isDragging = false.

Add a `Float.clamped(to:)` extension if not already present:
```swift
private extension Float {
    func clamped(to range: ClosedRange<Float>) -> Float {
        min(max(self, range.lowerBound), range.upperBound)
    }
}
```

**ViewModeControl.swift** â€” `struct ViewModeControl: View` per UI-SPEC ViewModeControl component:

Parameters:
- `@Binding var selected: ViewMode`
- `let hasTranscript: Bool` â€” gates "Text" segment availability (VIEW-03)

Body â€” HStack of 4 segments inside a pill-shaped container:
- Container: `HStack(spacing: 0)` with `.frame(height: 44)`, `.background(.ultraThinMaterial)`, `.background(Theme.controlBg)`, `.clipShape(Capsule())` (22px corner radius = full pill)
- Each segment: `ForEach(ViewMode.allCases, id: \.self)` button
  - Selected: `Theme.accent` background capsule, `Color.white` text, `.font(.system(size: 13, weight: .semibold))`
  - Unselected: clear background, `Theme.dimmedText` text, `.font(.system(size: 13, weight: .regular))`
  - Text segment: `.disabled(!hasTranscript)`, `.opacity(hasTranscript ? 1.0 : 0.35)` when disabled (VIEW-03 â€” dimmed, no tap response when no transcript)
  - Each segment: `.padding(.horizontal, 16)`, `.frame(height: 44)`
  - Tap action: `selected = mode` with `withAnimation(.easeInOut(duration: 0.20))` (UI-SPEC interaction contract)

**TeleprompterView.swift** â€” `struct TeleprompterView: View` per UI-SPEC Screen 2 and Pattern 6:

Parameters:
- `let text: String` â€” transcript or script text
- `@Binding var isScrolling: Bool` â€” driven by RecordingViewModel.isRecording
- `@Binding var scrollSpeed: Double` â€” 1.0 to 4.0 multiplier, default 1.0
- `let isFullScreen: Bool` â€” true for text-only content (D-14), false for text overlay mode (D-12)

Two rendering modes based on `isFullScreen`:

**Full-screen teleprompter mode (D-13, D-14):**
- Background: `.black`
- Split text into lines/paragraphs
- `ScrollView { ScrollViewReader { ... } }` containing VStack of text lines:
  - Current line (tracked by `@State private var currentLineIndex: Int = 0`): `.font(.system(size: 28, weight: .semibold))`, `Color.white`, left accent bar (4px wide, `Theme.accent`)
  - Other lines: `.font(.system(size: 16, weight: .regular))`, `Theme.teleprompterDim` (50% white)
  - Horizontal padding: 24px
- Text shadow on all lines: `.shadow(color: .black.opacity(0.6), radius: 2, x: 0, y: 1)` (UI-SPEC typography note)
- Auto-scroll: `.onReceive(Timer.publish(every: 0.05, on: .main, in: .common).autoconnect())` â€” guard `isScrolling`, increment `currentLineIndex` based on `scrollSpeed`, call `proxy.scrollTo(lineID, anchor: .center)` wrapped in `withAnimation(.linear(duration: 0.05))`
- Speed control: small `Slider(value: $scrollSpeed, in: 1...4, step: 0.5)` labeled "Speed" positioned at bottom of view, 13pt semibold label

**Text overlay mode (D-12):**
- Semi-transparent band: `Theme.textOverlayBg` background (55% black)
- Same text rendering but smaller area (bottom third of screen)
- No speed control in overlay mode (scrolling is passive)
- Current line highlight same style but at 16pt body size

Both modes:
- Scroll pauses when `isScrolling` becomes false (D-13: sync with recording state)
- Scroll resets `currentLineIndex = 0` when text changes

Lines are identified by index for ScrollViewReader: each line gets `.id(index)`.
