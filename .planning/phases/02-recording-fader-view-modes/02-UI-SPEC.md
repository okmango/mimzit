# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Recording + Fader + View Modes.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — native iOS SwiftUI |
| Preset | not applicable |
| Component library | SwiftUI + UIKit (UIViewRepresentable bridges for CALayer) |
| Icon library | SF Symbols (system) |
| Font | SF Pro (system font — `.font(.system(size:weight:))` or SwiftUI text styles) |

Note: This is a native iOS app. shadcn and web component libraries are not applicable. All components are SwiftUI views with UIViewRepresentable bridges where CALayer access is required (CompositorView, CapturePreviewView).

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, label-to-value inline spacing |
| sm | 8px | Fader label spacing, compact element padding |
| md | 16px | Default element spacing, screen edge padding |
| lg | 24px | Section gaps, control group vertical spacing |
| xl | 32px | Bottom safe area inset above record button |
| 2xl | 48px | Minimum touch target height for fader track |
| 3xl | 64px | Record button diameter |

Exceptions:
- Record button: 72px diameter (oversized circular, center-bottom placement, thumb-reachable). Source: D-03.
- Fader thumb: 28px diameter, 44px minimum hit target (Apple HIG minimum touch target). Source: D-05/D-06, Claude's Discretion.
- View mode pill: 44px height, auto-width fitting 4 segments. Source: D-10.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16pt (SF Pro) | Regular (400) | 1.5 | Transcript text overlay on video (VIEW-02), teleprompter body text |
| Label | 13pt (SF Pro) | Medium (500) | 1.2 | Fader position labels (e.g. "REF", "MIX", "CAM"), view mode segment labels (Ref / Cam / Blend / Text) |
| Caption | 11pt (SF Pro) | Regular (400) | 1.3 | Recording duration timer, status indicators |
| Display | 28pt (SF Pro) | Semibold (600) | 1.2 | Teleprompter current-line highlight text |

Notes:
- Teleprompter non-current lines: 24pt Regular (400). Current line: 28pt Semibold (600). Source: D-14, Claude's Discretion.
- All recording screen text that overlays video must use `.shadow(color: .black.opacity(0.6), radius: 2, x: 0, y: 1)` for legibility against any video background.
- Only 2 weights used throughout: Regular (400) and Semibold/Medium (500/600). This matches the existing codebase pattern.

## Screens and Component Inventory

### Screen 1: RecordingView (full-screen)

Layout layers (bottom to top):
1. `CompositorView` — fills entire screen edge-to-edge. Black background. AVPlayerLayer (bottom sublayer) + AVCaptureVideoPreviewLayer (top sublayer). Source: D-01, Pattern 2 from RESEARCH.md.
2. Text overlay layer — `TeleprompterView` or transcript overlay, conditionally visible. Source: D-11, D-12.
3. Control overlay layer — semi-transparent panels that auto-hide after 3s. Source: D-04.

Control overlay structure (auto-hide layer):
- **Top:** `ViewModeControl` — pill-shaped segmented control, centered horizontally, 16px from safe area top. Segments: Ref | Cam | Blend | Text.
- **Center:** `RecordingTimerLabel` — elapsed time display (HH:MM:SS), centered horizontally. Visible only while recording.
- **Bottom panel:** `VStack(spacing: 8)` containing:
  - `FaderView` (video) — labeled "VIDEO", full width minus 32px horizontal padding.
  - `FaderView` (audio) — labeled "AUDIO", same width, smaller track height (6px vs 8px for video fader).
  - `HStack` with record button centered.
- Bottom panel background: `Color.black.opacity(0.45)` capsule/rounded-rect, 16px horizontal padding from screen edges, 16px above home indicator safe area.

### Screen 2: TeleprompterView (text-only mode overlay)

Used when `content.contentType == .text` or when the view mode is `.text` (for video/audio with transcript).

Two sub-modes:
- **Text-only mode (D-13, D-14):** Full-screen dark background, large centered text, auto-scroll, speed control visible. Camera records simultaneously (D-15).
- **Text overlay mode (D-12):** Transcript scrolls as overlay on top of reference video. Semi-transparent dark background band behind each text block.

Teleprompter layout (text-only mode):
- Background: `.black`
- Text: centered horizontally, `ScrollView` with `ScrollViewReader`
- Current line: 28pt Semibold, `Color.white`, highlighted with subtle left accent bar (4px wide, `Theme.accent`)
- Other lines: 24pt Regular, `Color.white.opacity(0.50)`
- Speed control: small `Slider` or `Stepper` (Claude's Discretion on exact control), positioned above the record button area, labeled "Speed"
- Margins: 24px horizontal padding

### Component: FaderView

Custom `UIViewRepresentable` or pure SwiftUI `GeometryReader` + `DragGesture` horizontal fader.

| Property | Value |
|----------|-------|
| Track height (video fader) | 8px |
| Track height (audio fader) | 6px |
| Track color | `Color.white.opacity(0.30)` |
| Filled track color | `Theme.accent.opacity(0.80)` |
| Thumb diameter | 28px |
| Thumb color | `Color.white` |
| Thumb shadow | `shadow(radius: 4, y: 2)` |
| Hit target | 44px minimum height (center-aligned on track) |
| Haptic positions | 0% (left), 50% (center), 100% (right) — `UIImpactFeedbackGenerator(.light)`. Source: D-09. |

Labels: Left anchor label = "REF", right anchor label = "CAM" (video fader) or "MIC" (audio fader). 11pt Caption Regular. `Color.white.opacity(0.70)`.

## Interaction Contracts

### Auto-Hide Controls (D-04)

- Controls (ViewModeControl, FaderViews, RecordButton area) visible at full opacity on screen entry.
- After 3 seconds with no interaction, fade to `opacity(0)` with `animation(.easeOut(duration: 0.4))`.
- Any tap on the compositor area or fader drag: fade back in with `animation(.easeIn(duration: 0.2))`, reset 3-second timer.
- Exception: faders remain visible at full opacity while `DragGesture.isActive == true`. Source: D-04.
- RecordingTimerLabel follows the same auto-hide behavior.

### Fader Drag (D-05, D-06, D-09)

- `DragGesture(minimumDistance: 0)` on fader track and thumb combined hit target.
- Position mapped to 0.0–1.0 Float, clamped.
- `CATransaction.setDisableActions(true)` wraps opacity updates to prevent implicit CALayer animations during drag.
- Haptic: `UIImpactFeedbackGenerator` prepared on `DragGesture.onChanged` first call, triggered at 0.0, 0.50 (±0.02 tolerance), 1.0 positions.
- Value updates synchronously on drag; no debounce.

### View Mode Transition (D-10, D-11)

- Tap on segment: immediate switch, no delay.
- Transition: `withAnimation(.easeInOut(duration: 0.20))` on opacity changes for overlay views (text overlay fade in/out).
- CALayer opacity (compositor layers): set directly without SwiftUI animation — use `CATransaction` for smooth GPU-composited transition over 0.15s (`CATransaction.setAnimationDuration(0.15)`).
- In "Text" mode: video fader label changes from "CAM" to "TEXT" to reflect its new function (D-11).

### Recording Start/Stop (D-03, REC-04)

- Tap record button → `RecordingViewModel.toggleRecording()`.
- Recording start: button transitions idle → recording with `withAnimation(.spring(response: 0.3))`.
- Pulsing outer ring animation starts on recording begin.
- Recording elapsed timer (`00:00`) appears centered above faders on recording start, hidden when idle.
- Recording stop: button transitions recording → idle, pulsing ring removed, timer hides.

### Teleprompter Auto-Scroll (D-13)

- Scroll starts when `RecordingViewModel.isRecording` transitions to `true`.
- Scroll pauses when `isRecording` becomes `false`.
- Speed: configurable 1x–4x (default 1x). "Speed" slider or stepper in text-only mode.
- Implementation: `Timer.publish(every: interval, on: .main, in: .common).autoconnect()` drives `ScrollViewReader.scrollTo(_:anchor:)`.

## States and Copywriting Contract

### Recording Screen States

| State | What User Sees |
|-------|---------------|
| Loading | `ProgressView()` centered on black screen, `Color.white` tint. Caption "Preparing camera..." |
| Camera permission denied | Full-screen error card (see below) |
| Ready to record | Compositor live, controls visible, record button idle (white) |
| Recording | Controls visible, record button red + pulsing, timer counting up, auto-hide timer active |
| Controls hidden | Only compositor visible, tap to reveal controls |
| Recording stopped | Brief — immediately transitions to ready state. No "saving" state in Phase 2 (Phase 3 adds persistence). |

### Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Start Recording" (not shown as a button — the record button icon conveys this) |
| Recording status | "Recording" (displayed in timer label area as small caption above timer) |
| Timer idle label | None — timer is hidden when not recording |
| Camera permission denied heading | "Camera Access Required" |
| Camera permission denied body | "Mimzit needs camera access to record your practice. Go to Settings to allow access." |
| Camera permission denied CTA | "Open Settings" |
| Camera unavailable (simulator) heading | "Camera Not Available" |

### Destructive Actions

| Action | Approach |
|--------|----------|
