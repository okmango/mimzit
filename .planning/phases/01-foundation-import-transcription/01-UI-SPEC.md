## Spacing Scale

iOS spacing uses points (pt), not pixels. All values are multiples of 4 pt.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4 pt | Icon-to-label gap, inline badge padding |
| sm | 8 pt | Intra-row element spacing, secondary padding |
| md | 16 pt | Default list row padding, section body padding |
| lg | 24 pt | Section header vertical spacing |
| xl | 32 pt | Screen-level vertical breathing room |
| 2xl | 48 pt | n/a for Phase 1 |
| 3xl | 64 pt | n/a for Phase 1 |

Exceptions:
- Touch targets for interactive controls: minimum 44 pt tall (Apple HIG requirement, applies to the Transcribe Audio button and action sheet items)
- Content type icon in library row: 32 pt wide Ă— 32 pt tall (fixed, not scaled by Dynamic Type)
- Video thumbnail in content detail: 16:9 aspect ratio, full-width minus md (16 pt) horizontal padding on each side

## Typography

Use SwiftUI Dynamic Type text styles exclusively. Do not hardcode point sizes. Map semantic roles to system text styles so the app respects user accessibility settings.

| Role | SwiftUI Style | Approx Size (default) | Weight | Line Height |
|------|--------------|----------------------|--------|-------------|
| Body | `.body` | 17 pt | Regular (400) | 1.4 (system default) |
| Label / secondary | `.subheadline` | 15 pt | Regular (400) | 1.35 |
| Heading (screen title) | `.headline` | 17 pt | Semibold (600) | 1.3 |
| Caption / metadata | `.caption` | 12 pt | Regular (400) | 1.3 |

Rules:
- Use exactly 2 font weights: Regular (400) and Semibold (600). No bold (700), no medium (500).
- `.headline` = semibold body. Use for section headers, content item title in list row, and transcript label.
- `.caption` = metadata only. Use for content duration, file type badge, date added.
- Never use `.largeTitle` or `.title` in Phase 1 â€” navigation titles are set by NavigationStack automatically.

## Color

This app is for public speaking coaching â€” the dominant mood is focused, professional, dark-capable. The color system uses semantic SwiftUI roles that adapt to light and dark mode automatically. Custom colors follow the whozit `Color(light:dark:)` init pattern from `Shared/Theme.swift`.

| Role | Light Mode | Dark Mode | Usage |
|------|-----------|-----------|-------|
| Dominant (60%) | `.systemBackground` (white) | `.systemBackground` (near-black) | All screen backgrounds, List rows |
| Secondary (30%) | `.secondarySystemBackground` (light gray) | `.secondarySystemBackground` (dark gray) | Form sections, grouped list insets, content detail card area |
| Accent (10%) | `#2E7DDE` (custom blue) | `#5AA0F5` (brighter blue for dark) | See reserved list below |
| Destructive | `.red` (system) | `.red` (system) | Delete content item only |

Accent reserved for:
1. The '+' (add content) button in the navigation bar â€” only interactive affordance on the library screen
2. The 'Transcribe Audio' button label and spinner tint when transcription is in-progress
3. The 'Save API Key' button when the API key field is non-empty
4. Active/selected state of the eye-toggle icon in the API key SecureField

Accent is NOT used for: list row backgrounds, section headers, content type icons, duration labels, or the 'Start Practice' button (which is disabled/grayed in Phase 1 since it is wired in Phase 2).

Semantic color notes:
- Content type icons use fixed semantic colors per type:
  - Video: `.systemBlue` (SF Symbol `film`)
  - Audio: `.systemPurple` (SF Symbol `waveform`)
  - Text/script: `.systemGreen` (SF Symbol `doc.text`)
- Transcription status badge:
  - No transcript: no badge
  - Transcribed: `.systemGreen` SF Symbol `checkmark.circle.fill` at caption size
  - In-progress: accent-colored `ProgressView` replacing the Transcribe Audio button

## Screen Inventory

### Screen 1: Content Library (home screen)

Navigation container: `NavigationStack` with `.navigationTitle("Library")` (large title, default iOS behavior).

Primary visual anchor: accent '+' button in navigation bar trailing position â€” the only interactive affordance on the library screen when content exists.

**Empty state** (zero imported items):
- Vertically centered in remaining space below nav bar
- SF Symbol: `film.stack` at 56 pt, `.secondary` foreground color
- Heading: "No reference content yet" (`.headline`, `.primary`)
- Body: "Tap + to import a video, audio file, or type a script." (`.body`, `.secondary`)
- No button in empty state body â€” the '+' in the nav bar is the only CTA

**Populated state**:
- `List` with no section headers
- Each row: `ContentItemRow` (see Component Inventory below)
- Swipe-to-delete on each row triggers destructive alert (see Copywriting)
- No search bar in Phase 1

**Navigation bar**:
- Trailing: `Button { } label: { Image(systemName: "plus") }` tinted accent, `.accessibilityLabel("Add reference content")`
- No leading items

### Screen 2: Content Detail / Preview

Presented as `.sheet` from list row tap. (Source: CONTEXT.md D-08)

Structure (top to bottom inside a `NavigationStack` inside the sheet):
1. Media preview area (16:9 video thumbnail with play overlay, audio waveform placeholder, or text preview)
2. Title (`.headline`) â€” auto-generated from filename or "Untitled Script"
3. Metadata row: type label + duration (`.caption`, `.secondary`)
4. Divider
5. Transcript section:
   - If no transcript: 'Transcribe Audio' button (accent label, `.bordered` button style)
   - If in-progress: `ProgressView` with label "Transcribing..." replacing button
   - If complete: transcript text in scrollable `.body` font, green checkmark + "Transcript ready" label above
6. Spacer
7. 'Start Practice' button â€” DISABLED in Phase 1, grayed `.primary` button style, label "Start Practice" (wired in Phase 2)

Sheet navigation bar:
- Title: content title truncated to 1 line
- Trailing: `Button("Done")` to dismiss
- `.navigationBarTitleDisplayMode(.inline)`

### Screen 3: Add Content Action Sheet

Presented via `.confirmationDialog` (iOS native action sheet) from '+' button tap.

Three options (in order):
1. "Import Video" â€” launches `PhotosPicker` filtered to videos
2. "Import Audio" â€” launches `.fileImporter` with `.audio` UTType
3. "Type a Script" â€” presents inline text editor sheet

Cancel option auto-included by `.confirmationDialog`.

### Screen 4: Script Entry Sheet

Full-screen sheet. Contains:
- `NavigationStack` with `.navigationTitle("New Script")`
- `TextEditor` full-height, `.body` font, placeholder: "Start typing your script..."
- Toolbar: trailing "Save Script" button (disabled until text is non-empty)
- Toolbar: leading "Cancel" button

### Screen 5: Settings

`NavigationStack` containing `Form`. Two sections:

Section "OpenAI API":
- Configured state: `HStack` with green checkmark + "â€˘â€˘â€˘â€˘â€˘â€˘â€˘â€˘â€˘â€˘â€˘â€˘â€˘â€˘â€˘â€˘" secondary text + "Clear API Key" destructive button below
- Unconfigured state: `SecureField` with eye-toggle + "Save API Key" button (accent, disabled when empty)
- Footer: "Your API key is stored securely in Keychain. Get one at platform.openai.com"

Section "About":
- Version row: key/value HStack with `.secondary` value text

Settings is accessed via a tab bar tab (`.gearshape.fill` SF Symbol).

### Screen 6: API Key Entry Prompt (lazy trigger)

When user taps 'Transcribe Audio' and no API key is configured: present a `.sheet` (not an alert) with the same SecureField entry UI as the Settings screen API section. Sheet title: "OpenAI API Key Required". Footer explains lazy prompting: "Required for AI transcription. Text-only content does not need a key." Save button dismisses and immediately retries transcription.

## Component Inventory

### ContentItemRow

Used in the Library list.

```
[type icon 32pt]  [title .headline]          [duration .caption .secondary]
                  [type label .caption .secondary]   [transcript badge (optional)]
```

- Type icon: colored SF Symbol (see color section), 32 pt image frame
- Title: `.headline`, 1 line truncated
- Type label: "Video" / "Audio" / "Script" in `.caption`, `.secondary`
- Duration: right-aligned, `.caption`, `.secondary`; omit for scripts
- Transcript badge: `.checkmark.circle.fill` in `.systemGreen` at `.caption` size, shown only when transcript exists

Row height: 60 pt nominal (driven by content, not hardcoded)

### TranscribeButton

Two states:

State 1 â€” Idle (no transcript):
- `Button("Transcribe Audio")` with `.bordered` button style
- Accent foreground
- SF Symbol `waveform.badge.sparkles` leading icon

State 2 â€” In-progress:
- `HStack` with `ProgressView()` (small, accent-tinted) + Text("Transcribing...")
- Not tappable (replaced, not overlaid)

State 3 â€” Complete (not a button):
- `Label("Transcript ready", systemImage: "checkmark.circle.fill")` in `.systemGreen`
- `.caption` font

### APIKeyField

Reuse pattern from `carufus_whozit/Whozit/Features/Settings/SettingsView.swift` verbatim:
- `SecureField("sk-...", text: $apiKey)` + eye-toggle `Button`
- `Button("Save API Key")` disabled when `apiKey.isEmpty`
- Configured state: green checkmark + masked text + "Clear API Key" destructive button

## Interaction Contracts

### Import Video flow
1. User taps '+' â†’ confirmation dialog
2. User taps "Import Video" â†’ `PhotosPicker` appears (system UI)
3. User selects video â†’ picker dismisses â†’ loading indicator on library screen (brief, covers copy-to-sandbox)
4. New row appears at top of list with animation (`.default` list insertion animation)

### Import Audio flow
1. User taps '+' â†’ confirmation dialog
2. User taps "Import Audio" â†’ `.fileImporter` sheet appears (system Files UI)
3. User selects file â†’ sheet dismisses â†’ new row appears at top

### Type Script flow
1. User taps '+' â†’ confirmation dialog
2. User taps "Type a Script" â†’ Script Entry sheet slides up
3. User types â†’ Save Script button activates
4. User taps "Save Script" â†’ sheet dismisses â†’ new row with Script type icon appears at top

### Transcription flow
1. User taps content row â†’ detail sheet slides up
2. If video/audio: "Transcribe Audio" button visible
3. User taps "Transcribe Audio":
   a. If no API key: API key sheet appears first. After save, transcription auto-starts.
   b. If API key present: button transitions immediately to `ProgressView` + "Transcribing..."
4. On success: ProgressView replaced by "Transcript ready" label + transcript text scrollable below
5. On failure: ProgressView replaced by "Transcribe Audio" button + inline error message below button (see Copywriting)

### Headphone detection
- Checked lazily: only surfaced before a practice session starts (Phase 2 will show this warning)
- In Phase 1: headphone detection is implemented in foundation code but no warning UI is shown (no recording happens yet)

### Delete content item
- Swipe left on list row â†’ red "Delete" affordance
- User taps "Delete" â†’ `.alert` confirmation appears (see Copywriting)
- On confirm: row removed with `.default` animation, file deleted from sandbox
